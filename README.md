# 本地培训考场会话管理器

一个基于 React + Express + SQLite (WASM) 的本地培训考场会话管理系统。支持完整的考试场次生命周期管理：建考场场次、导入学员、排座、签到、异常处理、强制换座、结果导出，并将所有操作记入审计日志。

## 技术栈

- **前端**: React 18 + TypeScript + Vite + TailwindCSS + Zustand + lucide-react + react-router-dom
- **后端**: Express + TypeScript + tsx
- **数据库**: sql.js (基于 WASM 的 SQLite，无需本机编译，数据存 `exam-manager.db`)
- **持久化**: 每次写操作后立即 flush 到磁盘，重启后数据完全一致

## 快速启动

```bash
# 1. 安装依赖
npm install

# 2. 启动前后端 (3001端口 (后端) + 5173端口 (前端)
npm run dev
```

启动后，访问 http://localhost:5173

## 默认账号

系统启动后自动种子数据已预置了两个操作员:

| 用户名      | 显示名    | 角色    | 权限
----------|-----------|---------|--------------------------------
`admin`     | 管理员   | admin   | 全部权限，含关闭异常、管理操作员
`operator1` | 操作员1 | operator | 正常操作，**不能**关闭异常、不能管理操作员

## 核心功能

### 1. 考场配置

- 增删改查，配置考场名称、座位行数、座位列数
- 删除时检查：被场次引用的考场不可删除

### 2. 场次配置

- 关联考场，设置名称、考试日期、开始时间
- 状态流转: `草稿 → 排座中 → 签到中 → 进行中 → 已结束

### 3. 学员名单导入

- 支持粘贴 CSV 格式文本 (每行: `报名号,姓名,单位`)
- **原子性保证**: 批内重复号、或与场次内已有记录重复号 → **整批拒绝**，HTTP 409 返回重复报名号列表，零写入
- 不会"部分导入"污染已有对话

### 4. 座位安排

- **自动排座**: 清空现有分配后随机打乱学员顺序，按行优先填充
- **强制换座**: 指定学员 + 目标座位 + 原因；目标被占用时自动交换两学员位置；记录变更历史
- **撤销换座**: 点击变更记录中的撤销按钮，精确恢复原位置；已撤销的不再重复撤销
- 所有变更写入审计日志

### 5. 签到管理

- 逐个标记 `已签到` / `缺勤`
- 顶部实时统计已签到/缺勤/未签到人数
- 记录操作人和时间

### 6. 异常处理

- 上报类型: 缺勤 / 作弊 / 设备 / 其他
- 可关联学员，填写描述
- **权限控制**: 只有 `admin` 角色能关闭异常；普通操作员关闭会被 403 拒绝
- 关闭时必须填写关闭原因，记录关闭人和时间

### 7. 结果导出

- 一键导出完整 JSON:

```json
{
  "session": { ...场次信息... },
  "room": { ...考场信息... },
  "students": [ { ...学员+座位号+签到状态 } ],
  "anomalies": [ { ...异常+关闭信息 } ]
}
```

### 8. 审计日志

- 所有关键操作 (建考场/建场次/导入/排座/签到/异常/换座/关闭) 均记录
- 可按场次过滤查看
- 含: 时间、操作人、动作、详情

## 主流程复现步骤

> 下面按界面手动操作步骤，覆盖 **建场次 → 导入名单 → 排座 → 签到 → 异常 → 导出** 完整链路。

### 准备: 登录

1. 浏览器打开 http://localhost:5173
2. 下拉选择 **管理员**，点 **登录**

### 步骤 1: 创建考场

1. 左侧菜单进入 **考场管理**
2. 点 **新增考场**
3. 填写:
   - 名称: `培训教室A`
   - 行数: `5`
   - 列数: `8`
4. 点 **创建** → 列表看到新考场 (总座位: 40)

### 步骤 2: 创建场次

1. 左侧菜单进入 **场次管理**
2. 点 **新增场次**
3. 填写:
   - 名称: `2026第一期培训考试`
   - 考场: 选 `培训教室A`
   - 考试日期: `2026-06-20`
   - 开始时间: `09:00`
4. 点 **创建** → 列表看到新场次，状态为 `草稿`
5. 点 **查看详情** 进入场次详情页

### 步骤 3: 导入学员名单

1. 进入 **学员名单** Tab
2. 点 **导入学员**
3. 在文本框粘贴 (或点「示例」填充) 以下数据:

```
2026001,张三,第一单位
2026002,李四,第二单位
2026003,王五,第一单位
2026004,赵六,第三单位
2026005,钱七,第二单位
2026006,孙八,第一单位
2026007,周九,第四单位
2026008,吴十,第三单位
```

4. 点 **确认导入** → 列表出现 8 条记录

### 步骤 4: 自动排座

1. 切换到 **座位安排** Tab
2. 点 **自动排座** → 座位网格被填充，每位学员分配到一个格子
3. 可以观察座位变更记录表为空

### 步骤 5: 签到

1. 切换到 **签到管理** Tab
2. 对 2026001 张三，点 **签到** → 状态变绿色已签到
3. 对 2026002 李四，点 **签到**
4. 对 2026003 王五，点 **缺勤** → 状态变红色缺勤
5. 顶部统计: 已签到 2 / 缺勤 1 / 未签到 5

### 步骤 6: 上报异常

1. 切换到 **异常处理** Tab
2. 点 **报告异常**
3. 填写:
   - 类型: `缺勤`
   - 学员: 选 `2026003 王五`
   - 描述: `联系不上，手机关机`
4. 点 **提交** → 列表出现一条待处理异常

### 步骤 7: 关闭异常 (管理员)

1. 在刚才的异常行点 **关闭** 按钮
2. 原因填写: `确认缺席，已通知其单位领导`
3. 点 **确认关闭** → 状态变绿色已关闭

### 步骤 8: 导出结果

1. 切换到 **导出结果** Tab
2. 点 **导出 JSON** → 浏览器下载 `session-X-export.json
3. 预览区可看到完整的场次、学员、异常汇总统计

## 失败链路复现

### 链路 A: 导入重复报名号 → 整批拒绝 + 零污染

1. 进入任一场次详情 → **学员名单** Tab (假设已导入 8 人)
2. 点 **导入学员**，粘贴:

```
2026009,新人A,XX单位
2026001,与现有重复,YY单位
```

3. 点 **确认导入**
4. 期望结果:
   - 顶部显示红色错误: **「报名号重复: 2026001」**
   - 学员列表**仍然只有 8 人**，2026009 也未被导入
   - 审计日志**没有**此次失败操作记录

### 链路 B: 普通操作员不能关闭异常

1. 先以 **管理员** 登录，在某场次报一个异常 (参考步骤 6)
2. 右上角 **退出登录**
3. 重新登录，下拉选 **操作员1**
4. 进入同一场次 → **异常处理** Tab
5. 在刚才的异常行，**关闭**按钮应为灰色不可点击，hover 提示「仅管理员可关闭异常」
6. (可选) 直接用 curl 验证: 以 operator1 的 operatorId=2 请求 PATCH /api/sessions/X/anomalies/Y/close → 返回 403

### 链路 C: 强制换座后可撤销回原座位

1. 进入场次 → **座位安排** Tab (已自动排座)
2. 记录下 2026001 张三的当前座位号，例如 (0, 7)
3. 点 **强制换座**
4. 选择学员: `2026001 张三`，目标行 `4`，目标列 `7`，原因 `视力原因需要靠前`
5. 点 **确认换座**
6. 座位网格中张三移动到 (4, 7)，座位变更表新增 1 条记录
7. 在变更记录行点 **撤销**
8. 期望结果:
   - 张三回到原座位 (0, 7)
   - 变更记录显示「已撤销」灰色徽章，撤销按钮消失
   - 审计日志新增两条: force_change_seat 和 undo_seat_change

## 重启一致性验证

1. 完成上面主流程后，停止服务器 (Ctrl+C)
2. 确认项目根目录存在 `exam-manager.db`
3. 重新启动 `npm run dev`
4. 重新登录，进入同一场次查看:

| 检查项 | 期望一致 |
|--------|----------|
| 学员名单 | 人数、报名号、姓名相同 |
| 座位安排 | 每个学员的座位号 (row, col) 完全相同 |
| 签到管理 | 已签到/缺勤状态和操作人、时间一致 |
| 异常处理 | 异常内容、关闭人、关闭原因、状态一致 |
| 座位变更 | 变更记录、撤销状态一致 |
| 导出结果 | 导出 JSON 的字段和值完全一致 |
| 审计日志 | 所有历史操作记录完整 |

**实现原理**: 每次数据库写操作 (INSERT/UPDATE/DELETE) 完成后，立即调用 `sql.js` 的 `db.export()` 将整个数据库序列化并 `fs.writeFileSync` 写入磁盘。重启时从文件恢复。

## API 列表速查

所有写操作需要 `X-Operator-Id` 请求头。

### 操作员

| 方法 | 路径 | 鉴权 | 说明
|------|------|------|-----
| GET | `/api/operators` | 公开 | 列出操作员 (登录用)
| POST | `/api/operators` | 需登录 | 新增操作员
| POST | `/api/operators/login` | 公开 | 登录 (无密码，用户名即可)

### 考场 / 场次

| 方法 | 路径 | 说明
|------|------|-----
| GET/POST/PUT/DELETE | `/api/rooms[/:id]` | 考场 CRUD
| GET/POST | `/api/sessions` | 场次列表/创建
| GET | `/api/sessions/:id` | 场次详情
| PATCH | `/api/sessions/:id/status` | 更新状态

### 学员 / 座位 / 签到

| 方法 | 路径 | 说明
|------|------|-----
| GET | `/api/sessions/:id/students` | 学员列表
| POST | `/api/sessions/:id/students/import` | 导入，**重复号 409 整批拒绝**
| DELETE | `/api/sessions/:id/students/:sid` | 删除无座学员
| GET | `/api/sessions/:id/seats` | 座位图
| POST | `/api/sessions/:id/seats/auto` | 自动排座
| POST | `/api/sessions/:id/seats/force-change` | 强制换座
| POST | `/api/sessions/:id/seats/undo-change` | 撤销换座
| GET | `/api/sessions/:id/seats/changes` | 换座历史
| GET | `/api/sessions/:id/checkins` | 签到列表
| POST | `/api/sessions/:id/checkins` | 签到/缺勤

### 异常 / 审计 / 导出

| 方法 | 路径 | 说明
|------|------|-----
| GET/POST | `/api/sessions/:id/anomalies` | 列出/上报异常
| PATCH | `/api/sessions/:id/anomalies/:aid/close` | **仅管理员**关闭异常，非管理员 403
| GET | `/api/audit-logs` | 审计日志，?sessionId=X 过滤
| GET | `/api/sessions/:id/export` | 导出完整 JSON
