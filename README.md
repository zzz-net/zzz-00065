# 本地培训考场会话管理器

基于 Electron 的本地桌面应用，支持完整的考试场次生命周期管理：建考场场次、导入学员、排座、签到、异常处理、强制换座、结果导出，并将所有操作记入审计日志。

## 技术栈

- **桌面壳**: Electron (主进程) + 内嵌 Express 后端
- **前端**: React 18 + TypeScript + Vite + TailwindCSS + Zustand + lucide-react + react-router-dom
- **后端**: Express + TypeScript + tsx
- **数据库**: sql.js (WASM-SQLite，无需本机编译，数据存用户数据目录)
- **持久化**: 每次写操作后立即 flush 到磁盘，关闭再打开数据完全一致

## 使用方式

### 方式一：桌面应用启动 (推荐)

```bash
# 1. 安装依赖
npm install

# 2. 构建前端 + 编译 Electron 主进程
npm run build:all

# 3. 启动桌面应用
npx electron .
```

双击运行后，会自动弹出桌面窗口，无需手动打开浏览器或输入地址。

关闭窗口后所有数据自动保存到用户数据目录 (`%APPDATA%/exam-session-manager/exam-manager.db`)，再次打开数据完整恢复。

### 方式二：开发模式 (浏览器调试)

```bash
# 启动 Vite 前端 (5173) + Express 后端 (3001)
npm run dev
```

开发模式下打开 http://localhost:5173 ，数据保存在项目目录的 `exam-manager.db`。

### 方式三：打包为可分发桌面程序

```bash
npm run build:all
npx electron-builder --win
```

打包产物在 `release/` 目录下，为独立的 `.exe` 可执行文件，可直接分发给用户使用，无需安装 Node.js。

## 默认账号

| 用户名      | 显示名    | 角色     | 权限
|----------|-----------|---------|----------------------------------------
| `admin`     | 管理员   | admin   | 全部权限，含强制换座、撤销换座、关闭异常、管理操作员
| `operator1` | 操作员1 | operator | 签到/缺勤/上报异常/导入学员，**不能**强制换座、撤销换座、关闭异常、管理操作员

## 核心功能

### 1. 考场配置

- 增删改查，配置考场名称、座位行数、座位列数
- 删除时检查：被场次引用的考场不可删除

### 2. 场次配置

- 关联考场，设置名称、考试日期、开始时间
- 状态流转: `草稿 → 排座中 → 签到中 → 进行中 → 已结束

### 3. 学员名单导入

- 支持粘贴 CSV 格式文本 (每行: `报名号,姓名,单位`)
- **原子性保证**: 批内重复号、或与场次内已有记录重复号 → **整批拒绝**，HTTP 409，零写入
- 不会"部分导入"污染已有会话

### 4. 座位安排

- **自动排座**: 清空现有分配后随机打乱学员顺序，按行优先填充
- **强制换座** (仅管理员): 指定学员 + 目标座位 + 原因；目标被占用时自动交换；记录变更历史
- **撤销换座** (仅管理员): 精确恢复原位置；已撤销的不再重复撤销
- 普通操作员调用强制换座或撤销接口返回 403

### 5. 签到管理

- 逐个标记 `已签到` / `缺勤`
- 实时统计已签到/缺勤/未签到人数

### 6. 异常处理

- 上报类型: 缺勤 / 作弊 / 设备 / 其他
- **权限控制**: 只有 `admin` 能关闭异常，普通操作员 403 被拒

### 7. 结果导出

- 一键导出完整 JSON (场次 + 考场 + 学员含座位和签到 + 异常)

### 8. 审计日志

- 所有关键操作均记录 (含操作人、时间、详情)

## 主流程复现步骤

> 以管理员身份登录，覆盖 **建场次 → 导入名单 → 排座 → 签到 → 异常 → 导出** 完整链路。

### 准备: 启动并登录

1. 运行 `npx electron .` 启动桌面应用 (或 `npm run dev` 打开浏览器)
2. 下拉选择 **管理员**，点 **登录**

### 步骤 1: 创建考场

1. 左侧菜单 **考场管理** → **新增考场**
2. 名称: `培训教室A`，行数: `5`，列数: `8` → **创建**

### 步骤 2: 创建场次

1. 左侧菜单 **场次管理** → **新增场次**
2. 名称: `2026第一期培训考试`，考场: `培训教室A`，日期/时间按需 → **创建**
3. 点 **查看详情** 进入场次详情页

### 步骤 3: 导入学员

1. **学员名单** Tab → **导入学员**
2. 粘贴 (或点示例):

```
2026001,张三,第一单位
2026002,李四,第二单位
2026003,王五,第一单位
2026004,赵六,第三单位
2026005,钱七,第二单位
2026006,孙八,第一单位
2026007,周九,第四单位
2026008,吴十,第三单位
```

3. **确认导入** → 列表出现 8 条记录

### 步骤 4: 自动排座

1. **座位安排** Tab → **自动排座** → 座位网格被填充

### 步骤 5: 签到

1. **签到管理** Tab → 逐个点 **签到** 或 **缺勤**

### 步骤 6: 上报 + 关闭异常

1. **异常处理** Tab → **报告异常** → 填写类型、学员、描述 → **提交**
2. 管理员点 **关闭** → 填写原因 → **确认关闭**

### 步骤 7: 强制换座 + 撤销

1. **座位安排** Tab → **强制换座** (按钮仅管理员可见)
2. 选学员、目标座位、原因 → **确认换座**
3. 变更记录行点 **撤销** → 精确恢复原位置

### 步骤 8: 导出

1. **导出结果** Tab → **导出 JSON** → 下载完整数据文件

## 失败链路复现

### 链路 A: 导入重复报名号 → 整批拒绝 + 零污染

1. 场次详情 → **学员名单** Tab (已导入 8 人) → **导入学员**
2. 粘贴含重复号的内容:

```
2026009,新人A,XX单位
2026001,与现有重复,YY单位
```

3. **确认导入** → 红色错误提示「报名号重复: 2026001」
4. 学员列表**仍只有 8 人**，2026009 未被导入

### 链路 B: 普通操作员不能关闭异常

1. 管理员上报一个异常
2. **退出登录** → 以 **操作员1** 登录
3. 进入同一场次 → **异常处理** Tab
4. **关闭**按钮灰色不可点击，hover 提示「仅管理员可关闭异常」
5. (API 验证: `PATCH /api/sessions/X/anomalies/Y/close` + `X-Operator-Id: 2` → 403)

### 链路 C: 普通操作员不能强制换座

1. 以 **操作员1** 登录
2. **座位安排** Tab → **强制换座** 按钮灰色不可点击，hover 提示「仅管理员可强制换座」
3. 变更记录中 **撤销** 按钮灰色不可点击，hover 提示「仅管理员可撤销换座」
4. (API 验证: `POST /api/sessions/X/seats/force-change` + `X-Operator-Id: 2` → 403)
5. 座位数据**未被修改**

### 链路 D: 管理员强制换座后可撤销回原座位

1. 以 **管理员** 登录 → 场次详情 → **座位安排** Tab
2. 记录某学员当前座位号，如 (0, 7)
3. **强制换座** → 选该学员，目标 (4, 7)，原因 `视力原因` → **确认换座**
4. 座位网格中该学员移动到 (4, 7)，变更表新增记录
5. 变更记录行点 **撤销**
6. 学员回到原座位 (0, 7)，变更记录显示「已撤销」

## 重启一致性验证

1. 完成上面主流程后，**关闭桌面应用窗口**
2. 再次运行 `npx electron .` 启动
3. 重新登录，进入同一场次查看:

| 检查项 | 期望一致 |
|--------|----------|
| 学员名单 | 人数、报名号、姓名相同 |
| 座位安排 | 每个学员的座位号 (row, col) 完全相同 |
| 签到管理 | 已签到/缺勤状态和操作人、时间一致 |
| 异常处理 | 异常内容、关闭人、关闭原因、状态一致 |
| 座位变更 | 变更记录、撤销状态一致 |
| 导出结果 | 导出 JSON 的字段和值完全一致 |
| 审计日志 | 所有历史操作记录完整 |

**实现原理**: 数据库文件存于 Electron `app.getPath('userData')` 目录。每次写操作后立即 `db.export()` + `fs.writeFileSync` 写入磁盘。关闭窗口后数据自动持久化，再次打开从文件恢复。

## API 列表速查

所有写操作需要 `X-Operator-Id` 请求头。

| 方法 | 路径 | 权限 | 说明
|------|------|------|-----
| GET | `/api/operators` | 公开 | 列出操作员
| POST | `/api/operators` | 登录 | 新增操作员
| POST | `/api/operators/login` | 公开 | 登录
| GET/POST/PUT/DELETE | `/api/rooms[/:id]` | 登录 | 考场 CRUD
| GET/POST | `/api/sessions` | 登录 | 场次列表/创建
| GET | `/api/sessions/:id` | 登录 | 场次详情
| PATCH | `/api/sessions/:id/status` | 登录 | 更新状态
| GET | `/api/sessions/:id/students` | 登录 | 学员列表
| POST | `/api/sessions/:id/students/import` | 登录 | 导入 (重复号 409)
| GET | `/api/sessions/:id/seats` | 登录 | 座位图
| POST | `/api/sessions/:id/seats/auto` | 登录 | 自动排座
| POST | `/api/sessions/:id/seats/force-change` | **admin** | 强制换座
| POST | `/api/sessions/:id/seats/undo-change` | **admin** | 撤销换座
| GET | `/api/sessions/:id/seats/changes` | 登录 | 换座历史
| GET/POST | `/api/sessions/:id/checkins` | 登录 | 签到
| GET/POST | `/api/sessions/:id/anomalies` | 登录 | 异常列表/上报
| PATCH | `/api/sessions/:id/anomalies/:aid/close` | **admin** | 关闭异常
| GET | `/api/audit-logs` | 登录 | 审计日志
| GET | `/api/sessions/:id/export` | 登录 | 导出 JSON
