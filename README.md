# 培训考场管理系统（桌面版）

基于 Electron 封装的可交付 Windows 桌面应用。完整的考试场次生命周期管理：建考场、导入学员、自动排座、强制换座+撤销、签到、异常处理、一键导出，全部操作记入审计日志。

## ✨ 桌面版核心特性

### 🔌 开箱即用
- **双击即可打开**：安装后无需命令行、无需手动启动服务，启动时自动拉起内嵌 Express 后端并加载前端页面
- **独立于源码的数据目录**：运行数据（SQLite 数据库、配置文件）保存在操作系统用户目录，**不污染项目源码**
- **首次自动初始化**：首次启动时自动创建用户数据目录、写入配置文件、初始化空数据库

### 📁 用户数据目录

| 操作系统 | 默认数据目录 |
|---------|------------|
| Windows | `%APPDATA%/exam-session-manager` （如 `C:/Users/XXX/AppData/Roaming/exam-session-manager`） |
| macOS   | `~/Library/Application Support/exam-session-manager` |
| Linux   | `~/.config/exam-session-manager` |

目录内的关键文件：
```
exam-session-manager/
├── app-config.json          # 应用配置（端口、数据目录路径、最近会话ID、窗口尺寸）
├── exam-manager.db          # SQLite 数据库（考场、场次、学员、座位、签到、异常、日志）
└── Logs/                    # Electron 日志（可选）
```

> 💡 **换台机器数据不丢**：只要将上述目录完整拷贝到新机器的相同位置，重新打开应用即可恢复全部数据（座位映射、操作日志、导出记录 100% 一致）。也可通过「系统设置 → 数据目录」指定到 U 盘 / 网盘目录，实现便携。

### 🛡️ 启动容错

启动时自动执行以下检查，**任何一项失败都不会静默崩掉**，而是在专门的错误页显示：

| 检查项 | 失败时的表现 | 解决入口 |
|-------|-------------|---------|
| ✅ 数据目录可写？ | 显示「数据目录不可写」错误页，并说明具体权限原因 | 错误页提供「重试启动」和「系统设置」按钮 |
| ✅ 端口可用？   | 默认端口 3001 若被占用 → 自动尝试后续 50 个端口找到可用的；若全部失败 → 错误页 | 「系统设置」改端口 |
| ✅ 依赖完整？   | 启动后端进程失败 / 模块缺失 → 错误页，附带日志详情 | 执行 `npm install` |
| ✅ 30 秒内就绪？ | 后端启动超时 → 错误页 | 点「重试启动」 |

### 🕒 最近会话入口

- 首页 **Dashboard** 顶部显示最近访问的 5 个场次，并标记「上次访问」
- 每次打开 / 操作某个场次，自动记录到配置文件的 `recentSessionId`
- 重启应用后首页会高亮上次离开的场次，一键点击回到工作位置

## 🚀 使用方式

### 方式一：桌面应用启动（推荐给最终用户）

#### 首次启动流程

```bash
# 1. 首次使用：安装依赖 + 完整构建
npm install
npm run build:all

# 2. 启动桌面应用
npx electron .
```

**首次启动自动完成：**
1. ✅ 检查 `%APPDATA%/exam-session-manager` 目录，不存在则创建
2. ✅ 写入默认配置 `app-config.json`
3. ✅ 在该目录下初始化空数据库 `exam-manager.db`（自动建表 + 插入两个默认账号）
4. ✅ 检测 3001 端口是否可用，不可用则向后寻找
5. ✅ 启动 Express 后端（30 秒超时检测）
6. ✅ 创建桌面窗口并自动跳转到登录页

#### 日常启动

直接双击快捷方式（打包后）或再次执行 `npx electron .`，系统会：
- 自动打开上次保存的窗口位置和大小
- 自动加载已有数据库
- 首页高亮显示「上次访问」的场次

### 方式二：打包为独立 .exe（可分发）

```bash
npm run build:all
npx electron-builder --win portable
```

产物在 `release/培训考场管理系统 x.x.x.exe`，**无需用户安装 Node.js**，双击即用。数据目录依然在用户的 `%APPDATA%`。

### 方式三：开发模式（浏览器调试）

```bash
npm run dev
# 打开 http://localhost:5173
# 数据保存在项目目录下的 exam-manager.db
```

## 🔀 主流程（导入 → 排座 → 换座 → 撤销 → 导出）

每场考试在 4 步主流程面板中一键跳转（桌面版独有的快捷步骤栏）：

### 第 1 步：创建考场 + 场次

1. **考场管理** → 新增：`培训教室A`，5 行 × 8 列
2. **场次管理** → 新增：`2026第一期培训考试` → 选择教室 → 填日期时间 → 创建
   - ⚠️ **冲突拦截 1：同名场次**：名称+日期+开始时间三者完全相同的场次将被拒绝（HTTP 409），错误提示会写清冲突的字段，提供「跳转至已存在的场次」按钮

### 第 2 步：导入学员名单

1. 进入场次详情 → **桌面快捷步骤 1：导入学员名单** → 自动跳转 Tab
2. 点「导入学员」，粘贴 CSV：
   ```
   2026001,张三,第一单位
   2026002,李四,第二单位
   2026003,王五,第一单位
   2026004,赵六,第三单位
   2026005,钱七,第二单位
   2026006,孙八,第一单位
   2026007,周九,第四单位
   2026008,吴十,第三单位
   ```
3. 点「确认导入」
   - ⚠️ **冲突拦截 2：重复导入**
     - 本批内存在重复报名号：写清「报名号 xxx 出现在第 N 行、第 M 行」
     - 与场次已有学员报名号重复：写清「报名号 xxx（已有学员姓名）」与当前场次冲突
     - 两种冲突同时存在，**所有条目零写入**，不会「部分导入」污染会话
     - 错误消息末尾提示：「为避免污染数据，已全部拒绝，请修正后重新导入」

### 第 3 步：自动排座 → 强制换座 → 撤销

1. **桌面快捷步骤 2：自动排座** → 跳转 Tab → 点「自动排座」，座位网格瞬间填充
2. **桌面快捷步骤 3：强制换座/撤销**（仅管理员可见）→ 点「强制换座」
   - 选学员、填目标座位、写原因（如「视力原因」）→ 确认
   - 目标座位被占用时**自动互换**，并分别写入两条 `seat_changes` 记录
3. 在「换座记录」表格点 **撤销** → 学员精确回到原座位，记录标记「已撤销」
   - 已撤销的变更不可重复撤销（后端校验，HTTP 400 拦截）

### 第 4 步：签到 → 异常 → 导出

1. **桌面快捷步骤 4：签到/异常/导出** → 分别完成签到缺勤、上报异常（管理员可关闭）
2. **导出结果** Tab → 点「导出到文件...」
   - 弹出系统原生保存对话框（默认文件名：`session-{ID}-{场次名称}-{日期}.json`，非法字符自动过滤）
   - ⚠️ **冲突拦截 3：目标文件已存在** → 后端拒绝写入（HTTP 409），前端弹出系统原生警告框，显示文件路径、大小 KB、修改时间，按钮「取消 / 覆盖保存」，**不会悄悄覆盖**
   - 选择覆盖 → 重新带 `overwrite: true` 请求后端写入
   - 导出成功：绿色提示，显示路径 / 字节数 / 学员数 / 异常数
   - 每次文件导出操作都会写入审计日志 `export_session_file`

## 🛠️ 技术栈

- **桌面壳**: Electron 42（主进程 + preload + IPC）
- **内嵌后端**: Express 4 + TypeScript + sql.js（SQLite WASM，无需本机编译）
- **前端**: React 18 + Vite + TailwindCSS + Zustand + react-router-dom + lucide-react
- **打包**: electron-builder（portable 单文件 .exe）

## 🔐 默认账号

| 用户名 | 显示名 | 角色 | 权限 |
|-------|--------|------|------|
| `admin` | 管理员 | admin | 全部权限（强制换座、撤销换座、关闭异常、管理操作员、修改系统设置） |
| `operator1` | 操作员1 | operator | 建考场/场次、导入、排座、签到、报异常；**不能**强制换座、撤销、关异常、改设置 |

## 📜 API 列表补充（桌面版新增）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/system/info` | 获取数据目录路径、DB 大小、计数统计 |
| POST | `/api/system/data-directory/check` | 检查目录是否可写（自动创建 + 测试写入） |
| POST | `/api/system/data-directory/switch` | admin 切换数据目录（立即生效于内存，重启后生效于磁盘加载） |
| POST | `/api/sessions/:id/export/save` | 带 overwrite 字段保存 JSON 到本机路径，文件已存在返回 409 conflict |
| POST | `/api/export/check-path` | 预检查导出路径：文件存在？父目录可写？ |

**Electron IPC（preload 暴露）**：`config:get/set`、`dialog:selectDirectory/showSaveDialog/showMessageBox`、`system:checkPort/checkDirWritable`、`backend:restart/status`、`onBackendError/onBackendReady`、`shell:openDirectory`、`path:join/basename`。
